Important Concepts

Lesson 1: The Basics of the Web and HTML

How The Web Works

The world-wide web (www) is a network of computers that communicate with each other over a very wide area. The internet uses a protocol called HTTP to communicate with you, your browser and servers. When a user enters a website their computer sends an HTTP request to a server (computers that host the files that make up the web), which then finds and returns the relevant files. The user’s web browser then uses the returned files to display content to the user.

HTML

HTML stands for HyperText Markup Language. HTML documents make up most of the content on the web. They can contain text content, markup, references to other documents (images, videos etc) and links to other pages. HTML documents are made up of HTML elements, this refers to everything within a set of opening and closing tags.

HTML Tags

Markup consists of tags, which can also be nested. Some tags (called "void" tags) do not require a closing tag.

Computers Are Stupid

They cannot think for themselves and need to be told what to do very precisely by the programmer. They interpret instructions very literally and so simple errors, such as a missing “>” in a tag, can cause a program not to run correctly. Programmers therefore need to write exactly the way a computer understands (also known as writing with correct "syntax").

Inline vs. Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Lesson 2: Creating a Structured Document with HTML

HTML vs. CSS

HTML provides the structure of websites, whilst Cascading Style Sheets (CSS) describe the look and formatting of a document written in a markup language. The browser turns HTML tags into elements which form a tree. This tree-structure exists because HTML elements can have other elements inside of them. The structure is shown in the HTML document using indentation.

Document Object Model (DOM)

When programmers mention the "DOM", they are referring to the Document Object Model, which refers to the tree-like structure of a page. DOM is a cross-platform and language-independent method for representing and interacting with objects in HTML. The nodes of every document are organized into a tree structure, which is known as the DOM tree.

Developer Tools

Developer tools can be used to investigate the structure of existing websites, which is displayed using this tree-like structure.

Everything Is A Box

Everything on a webpage is actually a box. When designing a webpage, move from the big boxes down to smaller boxes, leaving minor details until last. Boxes should be defined using div tags. To apply the appropriate style to each one, it is best to give each element a class attribute (these are just like labels on boxes when moving home). You should use class names that make sense to you in order to better understand the purpose of the code.

Lesson 3: Adding CSS Style to HTML Structure

Linking to CSS Document

You should link the CSS document to the html document within the head tag, this makes the CSS easier to maintain. The format for this should be:
<head>
<link rel="stylesheet" href="css_document.css">
</head>
You can also put the style information between style tags within the head section, although this should usually only be used for very small projects.

Avoiding Repetition

An important aspect of CSS is that it allows programmers to avoid repetition, and inheritance of style properties allows developers to write less CSS code. In the long run this should improve the maintainability of the code, and enables the programmer to make changes to the style much more quickly and efficiently. Two other key aspects were mentioned in the lectures: increasing consistency and minimizing errors. Consistency is improved because related groups of elements can have identical styling values applied to them by assigning them to the same class. Errors are also minimized for similar reasons; if styles were applied to each element seperately then the programmer would have to carry out this repetitive task each time they want to update styling. Humans are not particularly good at boring, repetitive tasks (although computers are!) and so they are likely to make a number of errors during this process.

Div vs. Span

A div is a block-level element, it takes up the full width of the page, pushes other content further down and is used to control layout. A span is an inline element used for things like text decoration, and doesn't affect the layout of other content. Divs and spans are discussed in detail in this video

CSS Selectors

You use a selector in CSS to define what elements of the page a particular style will apply to, you can also apply different styles to classes using the .class_name selector method. The format of the code to apply style is "selector {property: value;}".

IDs vs. Classes

IDs can be used to uniquely identify a single element within the webpage.Classes are used to style a group of elements, this reduces repetition within the code.

Code, Test, Refine

When developing a website it is best to follow a process of code, test, refine.
  1. Try to get the structure of the page right by visually boxifying it (create structure using divs etc)
  2. In order to see the structure better it is often useful to add a margin and a red solid border to all elements initially.
  3. Work on the tags (add heading tags etc)
  4. Think about the size of the boxes, where to position them on the screen and apply styles (from biggest to smallest)
  5. Polish smaller details until complete

Flexbox

Flexbox can be used to display multiple elements (including divs) inline with each other. To do this you define a flex container by changing the display attribute of the parent container in CSS to value flex (ie. display: flex;).

Box Model

Content is surrounded by padding, a border and a margin respectively. The width of an element is calculated by adding the width of the content, left border, left padding, right padding and right border together.

Other CSS Tips

  1. In CSS the most specific rule is applied to every element
  2. To help deal with sizing issues related to elements you should set sizes in terms of percentages rather than pixels, or set the box-sizing attribute to border-box for every element.
  3. When using images it is good practice to make a folder called images within your project folder.
